Tags

The tags are colour coded depending on whether they are mandatory and must be applied to all entries or whether they are optional with a default value being used if they are not present. New tags are bold.

Text

TagDefaultPurpose
AdjectiveA reference to the text files, which contain an entry containing the adjective used this religion
ButtonPath to the button to display for the religion
CivilopediaRefers to a tag in the Civ4GameText file which stores the Platypedia description for the religion
DescriptionA reference to the text files, which contain an entry describing this religion
FreeUnitClassNO_UNITCLASSThe first to found this religion receives a number of units of this class, see iFreeUnits
GenericTechButtonPath to the button to display for the religion on the main screen if the game option to pick religions is enabled
MovieFilePath to the movie file to be played when the religion is founded
MovieSoundReference to the sound file used when playing the movie
SoundReference to the sound file used when the religion spreads to a new city
TechButtonPath to the button to display for the religion in the tech chooser
TechPrereqNO_TECHThe tech that unlocks this religion
TypeThe internal name of this religion

Integer

TagDefaultPurpose
iFreeUnits0The number of units received by the first to found this religion, see FreeUnitClass
iOrderPriority0Has no meaning for religions
iSpreadFactor0Determines how likely the religion is to spread to other cities without missionaries or buildings
iTGAIndexThe index into the TGA file for the religion icon.

Compound (Multi-line)

TagDefaultPurpose
GlobalReligionCommercesNONEThe commerce gained by the holy city of this religion for each city, of any player, that has the religion
  <iGlobalReligionCommerce>0</iGlobalReligionCommerce>
HolyCityCommercesNONEThe commerce gained in the holy city of this religion
  <iHolyCityCommerce>0</iHolyCityCommerce>
StateReligionCommercesNONEThe commerce gained by every city that has this religion for which it is also the state religion
  <iStateReligionCommerce>0</iStateReligionCommerce>
