Tags

The tags are colour coded depending on whether they are mandatory and must be applied to all entries or whether they are optional with a default value being used if they are not present. New tags are bold.

Text

TagDefaultPurpose
AdvisorNO_ADVISORThe strategic path that training this unit will most benefit. Reference to Civ4AdvisorInfos.xml
BonusTypeNO_BONUSBonus that the city must have to train this unit
CaptureNO_UNITReference to another unit. If this unit is captured the capturing player will gain a unit of the referenced type instead of this unit
CivilopediaA reference to the text files, which contain the Platypedia entry for this unit
ClassThe class as defined in CIV4UnitClassInfos.xml. Class is the thing that holds, say, a Swordsman and a Praetorian together; each civilization can only have one of each class.
CombatNO_UNITCOMBATThe unit combat type of the unit. Reference to CIV4UnitCombatInfos.xml
DefaultUnitAIThe default AI routine that the unit uses. Reference to Civ4UnitAIInfos.xml
DescriptionA reference to the text files, which contains the name displayed for this unit
DomainThe domain that the unit moves in: Land, Sea, Water or Immobile. Reference to Civ4DomainInfos.xml
DomainCargoNO_DOMAINThe Domain (AIR, SEA, or LAND) of units that the unit can carry
FormationTypeThe formation that the unit adopts when displayed
HelpAdditional help text that will be displayed in the Platypedia
HolyCityNO_RELIGIONThe unit can only be built in the holy city of this religion
HotKeyThe key, that when pressed selects the unit.
InvisibleNO_INVISIBLEThe type of invisibility that the unit possesses. If the unit has an invisibility type that unit can only be seen by other units (or buildings) that are specially configured to be able to see the invisibility type. Reference to Civ4InvisibleInfos.xml
LeaderPromotionNO_PROMOTIONPromotion that the unit gives to another unit when attached as a warlord
MinCultureLevelNO_CULTURELEVELThe minimum culture level that a city must have in order to train this unit
ObsoleteTechNO_TECHA tech, which once researched by a player, prevents the training of this unit
PrereqBuildingNO_BUILDINGThis building must be present in the city to train the unit
PrereqCorporationNO_CORPORATIONThis corporation must be present in the city to train the unit
PrereqReligionNO_RELIGIONThis religion must be present in the city to train the unit
ReligionTypeNO_RELIGIONNot used in the DLL, but exposed to python for mod use there
SlaveSpecialistTypeN0_SPECIALISTThe type of slave that will be formed if the unit is captured by a slaver and the slave settled in a city. If there is no value here then the default slave specialist as defined in the GlobalDefineAlt SLAVERY_DEFAULT_SLAVE_SPECIALIST setting will be used
SpecialNO_SPECIALUNITReference to CIV4SpecialUnitInfos.xml
SpecialCargoNO_SPECIALUNITThe type of SpecialUnit that the unit can carry
StateReligionNO_RELIGIONThe player must have this religion as a state religion in order to train the unit
StrategySid's Tips entry that will also be displayed in the Platypedia
TypeThe internal name of this unit
UnitRangeTypeUNITRANGE_UNLIMITEDThe range that the unit is allowed to move

Integer

TagDefaultPurpose
iAIWeight0The preference given by the AI to this unit, generally for Unique Units
iAdvancedStartCost-1The cost to train one of these units in advanced start mode
iAdvancedStartCostIncrease0The additional cost per unit to train more than one in advanced start mode
iAirCombat0The base strength for air attacks. Used in FTTW for the strength of opportunity ranged attacks
iAirCombatLimit0Percentage of the maximum amount of air or opportunity ranged damage that this unit can give that is actually given. So a value of 50 means that the unit will only ever deal out 50% of the damage it can do based on its iComnbat value
iAirRange0The range at which an air unit it can operate or the range at which a ground unit can attack
iAirUnitCap0The number of air unit capacity slots the unit takes up -- should be 1 for most, if not all, air units
iAnimalCombat0The percent bonus that the unit gets against Animals, in attack and defense
iAsset0A rough guide to the value of the unit to a player. This is used when determining which units are the best attackers/defenders, it is also used to help determine the strength of a player.
iAutoXP0Unit will gain this XP at the end of every turn. Disabled units to not gain this.
iBaseDiscover0The base number of research points this unit can provide when discovering a technology, used in conjunction with iDiscoverMultiplier. The gamespeed also applies a multiplier on top of this value
iBaseHurry0The base number of hammers this unit can provide when hurrying a building, used in conjunction with iHurryMultiplier. The gamespeed also applies a multiplier on top of this value
iBaseTrade0The base golds this unit can provide when conducting a trade mission, used in conjunction with iTradeMultiplier. The gamespeed also applies a multiplier on top of this value
iBombRate0The base amount that the air unit will reduce a cities defence by when bombing
iBombardRate0The base amount that the unit will reduce a cities defence by when bombarding
iCargo0The number of units that the unit can carry
iChanceFirstStrikes0The number of First Strike Chances that the unit has
iCityAttack0The percent bonus that the unit gets when attacking cities
iCityDefense0The percent bonus that the unit gets when defending cities
iCollateralDamage0The amount of collateral damage (in percent) that the unit deals
iCollateralDamageLimit0The maximum amount of collateral damage that the unit can deal
iCollateralDamageMaxUnits0The greatest number of enemy units that the unit can inflict collateral damage upon
iCombat0The base combat strength of the unit
iCombatLimit0Percentage of the maximum amount of damage that this unit can give that is actually given. So a value of 50 means that the unit will only ever deal out 50% of the damage it can do based on its iComnbat value
iConscription0The conscription value of this unit. When determining which unit to conscript a city loops through all the units it can build and the one with the highest conscription value is chosen
iCost-1The cost in hammers of the unit
iCultureGarrison0The amount of cultural pressure having this unit in a city provides. This impacts the suppression rate of a conquered city and revolt resistance.
iDiscoverMultiplier0This value, multiplied by the teams population is added to the iBaseDiscover value to determine the total research a unit can provide
iDropRange0The range at which the unit can 'jump'/paradrop from a friendly city
iEspionagePoints0The number of espionage points gained when infiltrating a rival player
iEvasionProbability0For air and parachute units; the odds of evading interception
iExtraCost0The additional upkeep in gold that this unit costs each turn over and above the standard cost
iFirstStrikes0The number of automatic First Strikes that the unit has
iGreatJestDuration0The duration of any great jest that the unit performs. The unit requires a value in the iGreatJestHappiness tag for this to have an effect.
iGreatJestHappiness0The increase in happiness that a city receives as a result of any great jest this unit performs
iGreatWorkCulture0The culture this unit provides when creating a great work, the value is multiplied by the number of the Era the player is in and modified by the gamespeed
iHillsAttack0The percent bonus that the unit gets when attacking hills
iHillsDefense0The percent bonus that the unit gets when defending hills
iHotKeyPriority0If two objects have similar HotKey combinations then the one with the highest priority will take precedence
iHurryCostModifier0Modifier used as a divisor to determine the production gained when hurrying this unit
iHurryMultiplier0This value, multiplied by the city population is added to the iBaseHurry value to determine the total hammers a unit can provide
iInterceptionProbability0The percent chance a unit has of intercepting an enemy unit within range trying to carry out an air mission
iLeaderExperience0The amount of experience this unit gives to units in the same stack when leading as a warlord
iMaxAmmoTypeTier-1The maximum applicable ammunition type that this unit can upgrade to; -1 indicates no limit
iMaxWeaponTypeTier-1The maximum applicable weapon type that this unit can upgrade to; -1 indicates no limit
iMinAreaSize-1The number of tiles in the area the city is on in order to train this unit
iMinPopulation0The minimum population a city must have in order to train this unit
iMorale750A measure of the will of a combat unit to keep fighting in the face of losses. A unit with a morale < 100 has a chance of abandoning a battle and disbanding. The morale of a unit decreases with damage taken.
iMoves0The number of moves the unit has at the start of each turn. Each tile has a movement cost that is reduced from this value whenever a unit moves onto it, the combination of these values determines how many tiles a unit can move in the turn
iNukeRange-1The range at which the 'nuke' can be launched - missiles only
iOrderPriority0The priority of this unit when determining the order in which actions are performed. A unit with a high order priority will act first
iPower0Affects the amount that the AI factors it into the equation when determining your military power
iTradeMultiplier0This value, multiplied by the capital cities trade profits is added to the iBaseTrade value to determine the total gold a unit can provide in a trade mission
iWithdrawalProb0The percent chance a unit has of withdrawing from a battle if it is losing. Maximum Value is in GlobalDefines.xml
iWorkRate0The rate at which the unit improves tiles
iXPValueAttack0The value, in Experience Points, that this unit is worth when attacking
iXPValueDefense0The value, in Experience Points, that this unit is worth when defending

TagDefaultPurpose
bAltDown0If true, the HotKey is only used if the Alt key is also pressed
bAlwaysHostile0If true, is treated it as an enemy unit by both friends and foes
bAnimal0If true, this unit is an animal. Barbarian animals do not enter cultural borders and have basic AI routines, they also tend to stick to their natural habitats.
bCanMoveAllTerrain0If true, can move through terrain marked as impassible
bCanMoveImpassable0If true, can move through plots flagged as impassible, typically used for air units
bCityPacifier0If true, the unit will quell all unhappiness in a city whose plot it is on and the occupation timer will reduce faster.
bCounterSpy0If true, this unit will help detect any rival spying activities on the plot and may be able to prevent them succeeding
bCtrlDown0If true, the HotKey is only used if the Control key is also pressed
bDestroy0If true, can half a cities production, used by spies
bFirstStrikeImmune0If true, the unit is immune to first strikes
bFixedAI0If true, the unit cannot change out of the default AI
bFlatMovementCost0If true, ignores movement modifiers such as routes
bFood0If true, the unit is produced with food as well as hammers; a city does not generate food while building this unit
bFound0If true, the unit can found a new city
bGoldenAge0If true, the unit counts as a Great Person to start a Golden Age
bHiddenNationality0If true, unit appears as a barbarian to other players so can cause havoc without starting a war
bIgnoreBuildingDefense0If true, bypasses all defences provided by buildings when attacking a city
bIgnoreTerrainCost0If true, ignores movement penalties from terrain
bInvestigate0If true and you your unit is in a rival city, you may be able to look at the city details
bInvisible0If true, the unit is always invisible to rivals
bLineOfSight0If true, unit can only reveal the content in the tiles in the direction they are facing
bMechanized0If true, the unit is considered to be a machine rather than organic
bMilitaryHappiness0If true, counts as a Military Unit in the eyes of the rabble; contributes to the '+happy for each military unit in a city' civic modifier
bMilitaryProduction0If true, counts as a military unit for production bonuses/penalties
bMilitarySupport0If true, costs money to maintain as part of your military
bMilitaryTrade0If true, the unit may be part of a military trade agreement
bNoBadGoodies0If true, when this unit is entering a Tribal Village, the villagers will never be hostile
bNoCapture0If true, the unit cannot capture cities
bNoDefensiveBonus0If true, does not receive defensive bonuses
bNoRevealMap0If true, the unit does not reveal any tiles except the one it is standing on
bNukeImmune0If true, takes no damage from nukes
bOnlyDefensive0If true, the unit cannot attack other units
bPillage0If true, can pillage improvements
bPrereqBonuses0If true and the unit is a boat, it only be trained if one of the areas around the city has water bonuses in it. This is used for water working units so they are not built where there is nothing for them to do. If the unit is not a water unit then it cannot be trained if there are bonuses in the surrounding area! That looks like a bug to me!
bPrereqPower0If true, city must have power to train the unit
bPrereqReligion0If true, can only be trained in a city with a religion. Any religion will meet the prereq
bQuickCombat0If true, the unit never triggers the camera-angle change and other battle effects when combating another unit
bRenderAlways0Used by graphics engine in main exe
bRenderBelowWater0Used by graphics engine in main exe
bRivalTerritory0If true, can explore rival territory without triggering a war
bSabotage0If true, can pillage improvements. This is the spy equivalent of bPillage
bShiftDown0If true, the HotKey is only used if the Shift key is also pressed
bSingleBuild0If true, unit is killed after building an improvement. Used by slave units
bSlave0If true, is a slave
bSpy0If true, is a spy
bStealPlans0If true, the unit can see all a rival civs maps for 2 turns
bSuicide0If true, unit is always killed after combat
bWorkerTrade0If true, this unit may be part of a worker trade agreement
