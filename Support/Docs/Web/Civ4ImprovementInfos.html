Tags

The tags are colour coded depending on whether they are mandatory and must be applied to all entries or whether they are optional with a default value being used if they are not present. New tags are bold.

Text

TagDefaultPurpose
ArtDefineTagRefers to a tag in the Civ4ArtDefines_Improvement file
CivilopediaRefers to a tag in the Civ4GameText file which stores the Platypedia description for the improvement
DescriptionA reference to the text files, which contain an entry for this improvement
HelpAdditional help text that will be displayed in the Platypedia
ImprovementPillageNONEIf the improvement is pillaged, this lists what it is downgraded to. (Ex. A pillaged Cottage downgrades to a Hamlet.
ImprovementUpgradeNONEThe improvement to which this one is upgraded after iUpgradeTime is passed
PrereqTechNO_TECHThe tech that a players team must know in order to create this improvement. Counting towards the iUpgradeTime to upgrade can happen before this teach is researched.
TypeThe internal name of this Improvement
UpgradeTechNO_TECHThe tech that a players team must know in order to start counting towards the iUpgradeTime to upgrade the improvement
WorldSoundscapeAudioScriptRefers to the Audiodefines file for the ambient background music that is played when the tile is on screen

Integer

TagDefaultPurpose
iAdvancedStartCost-1Cost to build the improvement in Advanced Start mode
iAdvancedStartCostIncrease0Additional cost for each improvement built in Advanced Start mode
iAirBombDefense-1The defence of this improvement against attacks by air units. (it seems that if the strength of the air unit is equal to the airbombdefense, there is a 50-50 chance that it gets destroyed on attack.)
iAnimalSpawnRatePercentage0Percentile modifier in the rate that the improvement spawns animals
iAppearanceProbability0Percentage chance that the improvement will placed on each valid tile during map generation
iBarbarianSpawnRatePercentage0Percentile modifier in the rate that the improvement spawns barbarian units
iDefenseModifier0The defence percentage added to all units on the tile of the improvement
iFeatureGrowth0Chance modifier for spreading local terrain feature (aka Forest) to adjacent plots
iGoodyRange0The minimum distance between goody huts
iHappiness0The Happiness modifier that the improvement provides to the nearest city
iHealthChangePartPercent0xxxDESCxxx
iPillageGold0The base gold value that a unit gets if this improvement is pillaged
iTilesPerGoody0States how many tiles must exist per goody hut generated. (E.g If the value is 40, for every forty tiles in game, one goody hut is placed.)
iUpgradeTime0The time in turns that is needed on standard gamespeed until it upgrades to another improvement

Boolean

TagDefaultPurpose
bActsAsCity0If true, this improvement acts as a city when determining things such as unit healing rates
bCarriesIrrigation0If true, the improvement will count as irrigated so you can irrigate tiles next to it
bFreshWaterMakesValid0If true, the improvement always can be built next to a river or a lake
bGoody0If true, the improvement acts as a goody hut
bGraphicalOnly0If true, the improvement is not displayed in the Platypedia or WorldBuilder
bHillsMakesValid0If true, the improvement always can be built on a hill, regardless of anything else
bNoFreshWater0If true, the improvement cannot be built on a fresh water plot
bOutsideBorders0If true, the improvement can be built outside a players borders
bPeakMakesValid0If true, this improvement may be built on peaks
bPermanent0If true, the improvement cannot be destroyed
bRequiresFeature0If true, the improvement needs a feature on the plot or it will be removed
bRequiresFlatlands0If true, the improvement can on be built on flatlands
bRequiresIrrigation0if true, the improvement needs an irrigated tile next to it before it can be built
bRequiresRiverSide0If true, the improvement can only be built on a plot with a river flowing next to it
bRiverSideMakesValid0If true, the improvement always can be built next to a river
bSeaBridge0If true, this improvement can be constructed on water and connect two land tiles that are opposite each other, allowing land units to safely pass between.
bUseLSystem0Uses the L system,a art file,and is always 1.
bWater0If true, the improvement needs to be built on water
