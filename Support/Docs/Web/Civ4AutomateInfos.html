Tags

The tags are colour coded depending on whether they are mandatory and must be applied to all entries or whether they are optional with a default value being used if they are not present. New tags are bold.

Text

TagDefaultPurpose
AutomateNO_AUTOMATEAutomationType this automation will invoke
ButtonButton displayed in game
CommandNO_COMMANDThe command used to invoke this automation type. This is COMMAND_AUTOMATE for all existing automations
DescriptionText containgin the name of teh automation
HelpHelp text for the automation in the game
HotKeyThe key, that when pressed selects this object
HotKeyAltAn alternate key, that when pressed selects this object

Integer

TagDefaultPurpose
iHotKeyPriority0If two objects have similar HotKey combinations then the one with the highest priority will take precedence
iHotKeyPriorityAlt0If two objects have similar alternate HotKey combinations then the one with the highest priority will take precedence
iOrderPriority0The priority of this automation when determining the order in which actions are performed. An automation with a high order priority will be processed first

Boolean

TagDefaultPurpose
bAltDown0If true, the HotKey is only used if the Alt key is also pressed
bAltDownAlt0If true, the alternate HotKey is only used if the Alt key is also pressed
bCtrlDown0If true, the HotKey is only used if the Control key is also pressed
bCtrlDownAlt0If true, the alternate HotKey is only used if the Control key is also pressed
bShiftDown0If true, the HotKey is only used if the Shift key is also pressed
bShiftDownAlt0If true, the alternate HotKey is only used if the Shift key is also pressed
bVisible0If true, the automation is displayed in the UI
