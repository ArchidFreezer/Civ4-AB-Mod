Tags

The tags are colour coded depending on whether they are mandatory and must be applied to all entries or whether they are optional with a default value being used if they are not present. New tags are bold.

Text

TagDefaultPurpose
AdjectiveReferences the Text which contains the civilization adjective (eg: "American", or "English")
ArtDefineTagRefers to a tag in the Civ4ArtDefines_Civilization file
ArtStyleTypeSets the art style for the civilization. This is used when displaying city buildings.
CivilizationActionSoundThe sound set for the unit actions. The sound sets are defined in the AudioAudio3DScripts.xml file
CivilizationSelectionSoundThe sound set for the unit selections. The sound sets are defined in the AudioAudio3DScripts.xml file
CivilopediaRefers to a tag in the Civ4GameText file which stores the Platypedia description for the improvement
DefaultPlayerColorReferences the default color for that civilization, in the game. The possible values are defined in the \Assets\XML\Interface\CIV4PlayerColorInfos.xml file
DerivativeCivNONEEither NONE, or the civilization that is more likely to be created when a colony is split off.
DescriptionReferences the Text which contains the civilization description (eg: "American empire", or "English empire")
ShortDescriptionReferences the Text which contains the civilization short description (eg: "America", or "England")
TypeThe internal name of this Civilization
UnitArtStyleTypeSets the art style for the civilization units. This is used when displaying units, and allows, for example, China and Japan to be displayed as asiatic, while Egypt and Mali will be displayed as african. The possible values are defined in the \Assets\XML\Civilizations\CIV4UnitArtStyleTypeInfos.xml file

Boolean

TagDefaultPurpose
bAIPlayable1Specifies if a civilization can be played by an AI player
bPlayable1Specifies if a civilization can be played by a player (either human or AI)

Compound (Multi-line)

TagDefaultPurpose
AnarchyCivicsNONEUNUSED
  <CivicType></CivicType>
BuildingsNONESpecifies if a civilization uses unique buildings. It contains a list of Building Classes, and the correspondant Buildings enabled for that civilization. The possible values are defined: for Building Classes in the \Assets\XML\Buildings\CIV4BuildingClassInfos.xml file, for Building Types in the AssetsXMLBuildingsCIV4BuildingInfos.xml file
  <Building>
    <BuildingClassType></BuildingClassType>
    <BuildingType></BuildingType>
  </Building>
CitiesNONEThe list of city names to be used for that civilization. The names will be available in the specified order (for non barbarian civilizations) or in a random order (for barbarian cities).
  <City></City>
DisableTechsNONESpecifies the technologies this civilization cannot research, this would also cut off any techs that have this as a prereq
  <TechType></TechType>
FreeBuildingClassesNONESpecifies which building classes a civilization starts with in its cities
  <BuildingClassType></BuildingClassType>
FreeTechsNONESpecifies which techs a civilization starts with
  <TechType></TechType>
FreeUnitClassesNONESpecifies how many units a civilization starts with, and their types. It contains a list of unit classes, and a number for each of them
  <FreeUnitClass>
    <UnitClassType></UnitClassType>
    <iFreeUnits>0</iFreeUnits>
  </FreeUnitClass>
InitialCivicsNONESpecifies the civics this civilization starts in. There should be a civic defined for each CivicOptionType
  <CivicType></CivicType>
LeadersNONELists the leaders for this civilization
  <LeaderName></LeaderName>
UnitsNONESpecifies if a civilization uses unique units. It contains a list of Unit Classes, and the corresponding Units enabled for that civilization
  <Unit>
    <UnitClassType></UnitClassType>
    <UnitType></UnitType>
  </Unit>
