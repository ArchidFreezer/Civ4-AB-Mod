Tags

The tags are colour coded depending on whether they are mandatory and must be applied to all entries or whether they are optional with a default value being used if they are not present. New tags are bold.

Text

TagDefaultPurpose
ButtonPath to a .dds file that contains the icon for this civic
CivicOptionTypeThe internal reference to this civic
CivilopediaPedia text for this civic
DescriptionReference to the text that contains the civic name. The text may be defined in any of the files in \Assets\XML\Text
HelpAdditional help text that will be displayed in the Pedia
StrategyStrategy text for the civic
TechPrereqNO_TECHTech that must be known in order to run the civic
TypeThe internal reference to this civic
UpkeepNO_UPKEEPReference to an UpkeepType that defines the maintenance costs of the civic
WeLoveTheKingText shown when a city is on "We Love The King day"

Boolean

TagDefaultPurpose
bBuildingOnlyHealthy0If true, buildings don't contribute to unhealthiness in cities
bCreateBarbarians0If true, the player can convert normal units into barbarians
bEnableStarSigns0If true, star signs may impact the players cities
bMilitaryFoodProduction0If true, cities convert food into production when training military units
bNoCapitalUnhappiness0If true, the players capital city will never become unhappy
bNoCorporations0If true, corporations cannot be founded or spread and any existing have no effect
bNoForeignCorporations0If true, foreign corporations cannot be founded or spread and any existing have no effect
bNoForeignTrade0If true, foreign trade routes cannot exist
bNoNonStateReligionSpread0If true, only the state religion can be spread in your cities
bStateReligion0If true, a state religionmay be adopted. If you switch to a civic that doesn't allow a state religion, your state religion will be automatically switched to NO_RELIGION
bTribalConscription0If true, allows the civ to conscript additional units
bUnitRangeUnbound0If true, units with a maximum range have the range cap removed
bUnitTerritoryUnbound0If true, units that are normally restricted to the players territory may venture outside it
bUpgradeAnywhere0If true, units can be upgraded outside the player teams territory
