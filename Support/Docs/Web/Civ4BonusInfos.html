Tags

The tags are colour coded depending on whether they are mandatory and must be applied to all entries or whether they are optional with a default value being used if they are not present. New tags are bold.

Text

TagDefaultPurpose
ArtDefineTagRefers to a tag in the Civ4ArtDefines_Bonus file
BonusClassTypeNO_BONUSCLASSUsed to specify the category of the bonus. Must be a type from the file Civ4BonusClassInfos where a frequency for each bonus class can be stated
CivilopediaRefers to a tag in the Civ4GameText file which stores the Platypedia description for the bonus
DescriptionA reference to the text files, which contain an entry for this bonus
HelpAdditional help text that will be displayed in the Platypedia
TechCityTradeNO_TECHThe technology that enables the trading of this resource
TechObsoleteNO_TECHThe technology that obsoletes the resource
TechRevealNO_TECHThe technology that reveals the resource to the player
TypeThe internal name of this bonus

Integer

TagDefaultPurpose
iAIObjective0Defines how much the AI values the resource. If this is set units will try and protect the resource when on the defensive or use it when calculating whether they want to start a war. This is set to 0 for all existing bonuses.
iAITradeModifier0Defines how much the AI values the resource when trading
iConstAppearance0The chance that the resource will be placed on the map
iGroupRand0If iGroupRange is non-zero, this is the percentage chance that each tile within iGroupRange radius will contain that resource
iGroupRange0If non-zero, every time this resource is placed, all tiles within iGroupRange radius have an iGroupRand chance to have this resource
iHappiness0Change in happiness the resource provides
iHealth0Change in healthiness the resource provides
iMaxLatitude90Maximum latitude plots with this bonus can be
iMinAreaSize0Minimum needed size of island or continent to make appear the resource
iMinLandPercent0If a resource can be on land and ocean, this is the percentage that will be located on land
iMinLatitude0Minimum latitude plots with this bonus can be
iPlacementOrder0The order in which bonuses are placed, starting at zero and counting up. All of the zero resources will be placed before proceed to those with iPlacementOrder 1 and so on. If after the higher priority bonuses are placed there are no available squares left, then none of the lower priority bonuses will actually appear
iPlayer0Approximate occurrences per player, in percent (%), so 150 is about 1.5 per player
iTilesPer0The fixed number of additional occurrences per x tiles, so 120 would mean an additional occurrence every 120 tiles existing on the map
iUnique0This is the range in tiles that this resource will be unique, all tiles within this radius are invalid for another resource

Boolean

TagDefaultPurpose
bArea0If true and the map has more than one continent, this resource will only be placed on one continent instead of all possible areas
bFlatlands0If true, the resource may appear only on flat land
bHills0If true, the resource may appear only on hills
bNoRiverSide0If true, the resource can not appear next to a river
bNormalize0If true, bonus is used for starting location normalization (extra resources that are added to even out starting locations)
bPeaks0If true, the resource may appear on peaks
bUseLSystem0If true, the bonus uses the LSystem for graphics placement

Compound (Multi-line)

TagDefaultPurpose
FeatureBooleansNONEDefines a set of features that may exist on a plot for the bonus to appear
  <FeatureType></FeatureType>
FeatureTerrainBooleansNONEDefines which terrains that features defined in FeatureBooleans must be on in order for the bonus to appear
  <TerrainType></TerrainType>
RandsNONEUsed to randomise the number of a bonus to place on a map. A random number will be generated up to the range of each of these values and added to iConstAppearance when determining the number of bonuses to place on a map
  <iRandApp1>0</iRandApp1>
  <iRandApp2>0</iRandApp2>
  <iRandApp3>0</iRandApp3>
  <iRandApp4>0</iRandApp4>
TerrainBooleansNONEDefines which terrain types the resource may appear on
  <TerrainType></TerrainType>
YieldChangesNONEDefines the change in plot yields
  <iYieldChange>0</iYieldChange>
