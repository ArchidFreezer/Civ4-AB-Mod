Tags

The tags are colour coded depending on whether they are mandatory and must be applied to all entries or whether they are optional with a default value being used if they are not present. New tags are bold.

Text

TagDefaultPurpose
ArtDefineTagRefers to a tag in the Civ4ArtDefines_Features file
CivilopediaRefers to a tag in the Civ4GameText file which stores the Platypedia description for the feature
DescriptionA reference to the text files, which contain an entry for this feature
EffectTypeThe internal reference to an object instance of this type
HelpAdditional help text that will be displayed in the Platypedia
MovieDefineTagThe path to the moviw file to play when this feature is first seen by a player
OnUnitChangeToNO_FEATUREWhen any unit moves through the plot the feature is on it will change to the feature defined here
TypeThe internal name of this feature
WorldSoundscapeAudioScriptAudio played on close zooms for plots with the feature

Integer

TagDefaultPurpose
iAdvancedStartRemoveCost0Cost to remove the feature in advanced start
iAppearance0Chance out of 10000 that a feature will be added to a valid plot when generating the map. Setting this value to 10000 guarantees that the feature will exist wherever possible. If the value is exactly 10000 then this has a spoecial meaning to the map generator as it expects the feature to be a Flood Plain and will place it next to rivers if the terrain allows it, removing any bonuses that exist on the plot.
iDefense0Defensive bonus (percent) given to unit on a plot with the feature
iDisappearance0Chance that a feature will disappear naturally over time.
iEffectProbability0Chance for effect listed in EffectType to occur when a unit enters the plot with the feature
iGrowth0Modifier to the chance that a feature will expand to a neighbouring plot
iHealthPercent0Modifier to health of city with feature nearby; ex: iHeathPercent = -25 means that each plot with the feature causes .25 unhealthiness to the city
iMaxAdjacentWaterSize0xxxDESCxxx
iMaxLatitude0The maximum latitude that this feature may appear
iMinAdjacentWaterSize0xxxDESCxxx
iMinLatitude0The minimum latitude that this feature may appear
iMovement1Movement cost for units entering a plot due to the feature
iSeeThrough0The change in height that the feature causes. This goes towards determining whether a unit can see through the plot or not
iTurnDamage0Percentage of a units current hit points taken as damage on a plot with the feature each turn
iUniqueSize0For unique features te minimum number of tiles the feature requires
iWarmingDefense0Impact the feature has in reducing global warming

Boolean

TagDefaultPurpose
bAddsFreshWater0If true, the feature is counted as fresh water
bImpassable0xxxDESCxxx
bNoAdjacent0If true, the feature is allowed on a tile next to a tile with the same feature
bNoCity0If true, or not the feature removes the ability to found a city on the tile
bNoCoast0If true, the feature is allowed on a tile next to the coast
bNoImprovement0If true, or not the feature makes it impossible to build an improvement on the tile
bNoRiver0If true, the feature is allowed on a tile next to a river
bNukeImmune0If true, the feature is immune to destruction by nukes
bRequiresFlatlands0If true, the feature requires a flat tile (as in, not hills or peaks)
bRequiresRiver0If true, the feature must be next to a river
bUnique0If true, there will only be one of these features in the game
bVisibleAlways0If true, the feature is always visible on the map even if the player can no longer see the plot
